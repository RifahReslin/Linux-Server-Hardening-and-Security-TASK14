# Linux Server Hardening and Secure Configuration

This repository contains documentation for Task 14: Linux Server Hardening & Secure Configuration.
The task focuses on securing a Linux system by applying standard hardening techniques and security best practices.

## Objective
- Review default Linux system settings
- Remove unused user accounts
- Apply least privilege principle
- Secure SSH access
- Configure firewall rules
- Disable unnecessary services
- Protect sensitive system files
- Monitor system logs

## Tools Used
- Ubuntu / Kali Linux
- Lynis
- CIS Benchmarks

## Task Overview
The Linux system was hardened by restricting user access, securing SSH, updating packages, configuring firewall rules, and monitoring system logs to reduce security risks.

## Key Learnings
- Importance of server hardening
- Secure user and privilege management
- SSH security best practices
- Firewall configuration
- Log monitoring basics

## Outcome
This task improved practical knowledge of securing Linux servers against common attacks.
